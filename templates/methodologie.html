Méthodologie de Travail

Organisation et Processus

Une approche agile et collaborative pour développer Pharma App avec rigueur et efficacité.

Approche de développement

Méthode itérative et collaborative

  • Inspirée des méthodes agiles
  • Binôme permanent avec validation des étapes
  • Réunions client régulières à deux
  • Adjustements constants après chaque réunion

Validation et planification

  • Wireframes & MLD validés avant développement
  • Intégration continue avec tests locaux
  • Chaque fonctionnalité testée, validée, puis intégrée
  • Documentation systématique (.md)

Organisation d'équipe

  • Charles Massey - Lead développeur, architecte technique
  • Jean-François Poux - Développeur full-stack, wireframes, MLD
  • Complémentarité des compétences
  • Validation des étapes par le lead développeur

Workflow Git & GitHub

Organisation des branches

  • main → version stable et validée
  • dev → intégration des fonctionnalités

Cycle de développement

  • Développement et tests locaux
  • Pull request avec relecture de code
  • Tests avant merge dans main

Sécurité et qualité

  • Exclusion des fichiers sensibles (config.py)
  • Relecture systématique du code
  • Nettoyage des branches obsolètes
  • Aucune donnée sensible en clair dans le code

Outils et collaboration

Modélisation & Conception

  • Balsamiq - Wireframes et maquettes
  • Looping - Modèle Logique de Données (MLD)
  • Scripts SQL - Création et gestion de la base

Communication & Gestion

  • Notion / Email / Meet - Échanges et points réguliers
  • GitHub - Suivi des tâches et bugs
  • To-do agile - Gestion des priorités
